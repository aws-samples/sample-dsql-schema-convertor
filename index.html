About Aurora DSQL: Aurora DSQL is a serverless, distributed SQL database with PostgreSQL compatibility. It supports sequences (with explicit CACHE), identity columns, and SQL functions. However, it does not support PL/pgSQL, triggers, foreign keys, extensions, or temporary tables. This tool automatically converts your PostgreSQL schemas to be DSQL-compatible.

Aurora DSQL Key Constraints

⚠

Sequences Require CACHE

Sequences are supported but require explicit CACHE 1 or CACHE >= 65536. SERIAL/BIGSERIAL are converted to identity columns.

✕

No Foreign Keys

Foreign key constraints are not enforced. Referential integrity must be managed at the application level.

⚠

SQL Functions Only

Only LANGUAGE SQL functions are supported. PL/pgSQL procedures and functions must be moved to the application layer.

✕

No Triggers

Triggers cannot be created. Use application-level event handling instead.

✕

No Extensions

PostgreSQL extensions (e.g., PostGIS, pg_trgm) are not available. Use built-in functionality only.
