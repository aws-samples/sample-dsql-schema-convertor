About Aurora DSQL: Aurora DSQL is a serverless, distributed SQL database with PostgreSQL compatibility. It has specific constraints including no stored procedures, no triggers, no sequences (use UUIDs), no foreign keys enforced at the database level, and limited DDL transaction support. This tool automatically converts your PostgreSQL schemas to be DSQL-compatible.

Aurora DSQL Key Constraints

✕

No Sequences/SERIAL

Use gen_random_uuid() or application-generated IDs instead of SERIAL/BIGSERIAL.

✕

No Foreign Keys

Foreign key constraints are not enforced. Referential integrity must be managed at the application level.

✕

No Stored Procedures

Functions, procedures, and triggers are not supported. Move logic to the application layer.

✕

No Triggers

Triggers cannot be created. Use application-level event handling instead.

⚠

Limited DDL in Transactions

DDL statements cannot be mixed with DML in the same transaction.

✕

No Extensions

PostgreSQL extensions (e.g., PostGIS, pg_trgm) are not available. Use built-in functionality only.
